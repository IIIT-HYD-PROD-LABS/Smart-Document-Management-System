  <action>
**config.py overhaul:**

Replace the entire `Settings` class in `backend/app/config.py` with a hardened version:

1. Switch from `class Config:` to `model_config = SettingsConfigDict(env_file=".env", case_sensitive=True, extra="ignore")` (pydantic-settings v2 style).

2. Remove defaults from security-critical fields:
   - `SECRET_KEY: str` -- NO default. App crashes on startup if missing.
   - `DATABASE_URL: str` -- NO default. App crashes on startup if missing.

3. Change insecure defaults:
   - `DEBUG: bool = False` (was `True`)
   - `ACCESS_TOKEN_EXPIRE_MINUTES: int = 30` (was `60 * 24` = 1440)

4. Add new config fields needed by Phase 1 plans (so they exist when plans 01-02 and 01-03 run):
   - `REFRESH_TOKEN_EXPIRE_DAYS: int = 7`
   - `ALLOWED_ORIGINS: list[str] = ["http://localhost:3000"]`
   - `RATE_LIMIT_AUTH: str = "5/minute"`
   - `RATE_LIMIT_UPLOAD: str = "10/minute"`
   - `RATE_LIMIT_DEFAULT: str = "60/minute"`
   - `LOG_LEVEL: str = "INFO"`
   - `LOG_JSON_FORMAT: bool = True`

5. Add a `field_validator` on `SECRET_KEY` that rejects:
   - Known insecure values: `"super-secret-key-change-in-production"`, `"super-secret-docker-key"`, `"changeme"`, `"secret"`
   - Keys shorter than 32 characters
   - Raise `ValueError` with a helpful message.

6. Keep the `load_dotenv()` call at the top of the file (pydantic-settings loads .env automatically but explicit call ensures compatibility).

7. Keep the `os.makedirs` calls at the bottom for UPLOAD_DIR and MODEL_DIR.

8. Remove the old `class Config:` inner class.

**requirements.txt update:**

Update `backend/requirements.txt` to add new libraries and update existing ones:
- Add: `slowapi==0.1.9`, `structlog==25.5.0`, `asgi-correlation-id==4.3.4`
- Update: `PyJWT==2.8.0` -> `PyJWT==2.11.0`, `alembic==1.13.0` -> `alembic==1.18.4`, `pydantic-settings==2.1.0` -> `pydantic-settings==2.13.0`
- Keep all other dependencies unchanged.

**Create backend/app/utils/rate_limiter.py (NEW FILE):**

Create the rate limiter module so it's available for Wave 2 plans (01-02 and 01-03) to import:

```python
from slowapi import Limiter
from slowapi.util import get_remote_address
from app.config import settings

limiter = Limiter(
    key_func=get_remote_address,
    storage_uri=settings.REDIS_URL,
    default_limits=[settings.RATE_LIMIT_DEFAULT],
    headers_enabled=True,
)
```

This creates a module-level `limiter` instance that can be imported by any router. Plan 01-02 imports it for auth endpoint rate limiting, and Plan 01-03 imports it for upload endpoint rate limiting and wires it into the FastAPI app state.
