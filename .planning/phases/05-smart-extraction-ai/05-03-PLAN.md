1. Create `backend/app/schemas/settings.py`:
   ```python
   class LLMSettingsResponse(BaseModel):
       """Response schema -- never exposes full API key."""
       llm_provider: str = "openai"
       model_name: str | None = None
       api_key_set: bool = False        # True if user has an API key stored
       api_key_last4: str | None = None  # Last 4 chars of API key for display
       ollama_base_url: str | None = None

   class LLMSettingsUpdate(BaseModel):
       """Update schema -- API key is optional (only update if provided)."""
       llm_provider: str = Field("gemini", pattern="^(gemini|openai|anthropic|local)$")
       model_name: str | None = None
       api_key: str | None = Field(None, min_length=10, description="Full API key, will be encrypted before storage")
       ollama_base_url: str | None = None
   ```

2. Create `backend/app/routers/settings.py`:
   - `router = APIRouter(prefix="/api/settings", tags=["Settings"])`
   - `GET /api/settings/llm` -- Protected with `get_current_user`:
     - Query `UserLLMSettings` for current_user.id.
     - If no row exists, return default `LLMSettingsResponse()`.
     - If row exists, build response: provider, model_name, api_key_set=True/False, api_key_last4=last 4 chars of decrypted key (or None), ollama_base_url.
     - CRITICAL: Never return the full API key. Only `api_key_last4` (last 4 chars).
   - `PUT /api/settings/llm` -- Protected with `get_current_user`:
     - Accept `LLMSettingsUpdate` body.
     - Upsert: query existing row for user, create if not exists.
     - Update `llm_provider` and `model_name` always.
     - Update `api_key_encrypted` only if `body.api_key` is not None (use `encrypt_api_key()` method).
     - If provider is "local", set `ollama_base_url` from body. Otherwise set it to None.
     - Commit and return updated `LLMSettingsResponse`.
   - Import and use: `from app.utils.security import get_current_user`, `from app.database import get_db`, `from app.models.user_settings import UserLLMSettings`, `from app.schemas.settings import LLMSettingsResponse, LLMSettingsUpdate`.

3. Register the router in `backend/app/main.py`:
   - Add `from app.routers import auth, documents, ml, settings` (update existing import).
   - Add `app.include_router(settings.router)` after the ml router.
  </action>
