**Part A: Update search_documents in backend/app/routers/documents.py**

Change the FTS filter block to OR-combine FTS with trigram similarity. The OR-combine ensures typos hit via trigram even when FTS misses:

```python
from sqlalchemy import or_, func, Float
from sqlalchemy.dialects.postgresql import TSVECTOR

# In search_documents:
if len(q) <= 2:
    # Very short queries: ILIKE only (trigram requires min 3 chars)
    query = query.filter(Document.extracted_text.ilike(f"%{q}%"))
    rank_expr = None
else:
    search_query = func.plainto_tsquery("english", q)
    rank_expr = func.ts_rank(Document.search_vector, search_query)
    query = query.filter(
        or_(
            Document.search_vector.op("@@")(search_query),    # FTS: stemmed match
            Document.extracted_text.op("%")(q),               # trigram: typo tolerance
        )
    )

total = query.count()
if rank_expr is not None:
    documents = query.order_by(rank_expr.desc()).offset((page - 1) * per_page).limit(per_page).all()
else:
    documents = query.order_by(Document.created_at.desc()).offset((page - 1) * per_page).limit(per_page).all()
```

NOTE: The OR-combine handles short domain terms like "UPI", "GST" correctly — short exact terms hit via FTS even if trigram misses. Do NOT lower the pg_trgm similarity threshold globally — use the default 0.3.
